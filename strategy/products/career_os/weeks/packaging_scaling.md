# Career OS Module: Week 7 — Packaging & Scaling

Questo documento è il manuale operativo per la **Week 7**.
Obiettivo: Ora che hai il "Flagship Project" (W6), dobbiamo usarlo come ariete per sfondare nelle candidature.
Passiamo dalla modalità "Costruzione" alla modalità "Vendita Massiva".

---

## 1. Project Injection (CV & LinkedIn)

Il tuo progetto non deve stare solo su GitHub. Deve urlare dal tuo CV.

### A. CV Update (Sezione Projects)
Inserisci questo blocco subito dopo la tua esperienza lavorativa (o prima, se sei Junior).

**Format:**
> **[Nome Progetto] (Live Demo: mario-ai.web.app)** | *Next.js, Python, Firestore, OpenAI*
> *   **Built** an end-to-end RAG assistant for [Target User] that reduces research time by **80%**.
> *   **Architected** a scalable serverless backend on **GCP (Cloud Functions)** handling concurrent user requests.
> *   **Implemented** custom **Hallucination Guardrails** increasing answer accuracy to **95%** (verified vs baseline).

### B. LinkedIn Featured Section
1.  **Crea un Post** (usando lo Share Kit W4 ma aggiornato): Video Demo + Link al sito live.
2.  **Mettilo in Featured**: Deve essere la prima cosa che vedono.
3.  **Aggiorna l'Headline**: Aggiungi "Building [NomeProgetto]" se è rilevante.

---

## 2. The "Killer README" Template

Nessun Hiring Manager leggerà il tuo codice riga per riga. Leggeranno il README.
Se il README fa schifo, il progetto non esiste.

**Struttura Obbligatoria:**

1.  **Header Visivo**: Logo del progetto o screenshot della UI.
2.  **One-Liner**: "What is this?" (Chiaro, non tecnico).
3.  **Demo Link**: "Try it Live here -> [Link]" (Cruciale!).
4.  **Architecture Diagram**: Un diagramma (anche semplice, fatto con Mermaid o Excalidraw) che mostra: *Frontend -> API -> LLM -> Vector DB*. Di mostra che capisci i sistemi.
5.  **Key Features**: 3 bullet point (es. "Streaming UI", "Citation Tracking", "Auth").
6.  **Tech Stack**: Icone o lista (Next.js, Tailwind, GCP, LangChain).
7.  **How to Run**: Istruzioni per chi vuole clonarlo (npm install, env vars).

---

## 3. The "10-Application Sprint" (Protocollo Massivo)

Hai l'asset (Progetto). Hai i documenti (CV+README).
Ora facciamo i numeri. Questa settimana invii **10 candidature QUALITY**.

**Workflow Giornaliero (Lun-Ven):**
*   **Mattina (Scouting)**: Seleziona 2 JD dal tuo Job Feed (Tier 1 o Tier 2).
*   **Pranzo (Analisi)**: Leggi le JD, trova le keyword chiave mancanti nel tuo CV.
*   **Pomeriggio (Tailoring)**:
    1.  Clona il tuo CV Master.
    2.  Sposta il "Flagship Project" più in alto se la JD chiede quelle skill.
    3.  Modifica il Summary: "GenAI Engineer who built [NomeProgetto]..."
*   **Sera (Invio)**: Manda l'application.

**Checklist Qualità (Prima di premere Send):**
*   [ ] Il link al progetto nel CV funziona?
*   [ ] Hai usato le stesse parole chiave della JD nel Summary?
*   [ ] La Cover Letter (se richiesta) cita il progetto come "prova" delle skill?

---

## 4. Output della Week 7

*   [ ] **CV & LinkedIn Aggiornati**: Il progetto è visibile e cliccabile.
*   [ ] **Repo "Hiring-Ready"**: Il README ha il diagramma architetturale e il link alla demo.
*   [ ] **10 Candidature Inviate**: Tracciate nel Pipeline Tracker (W2) con status "Applied".

> **Gate Finale**: Se un recruiter guarda il tuo CV oggi, capisce che non sei solo un teorico ma uno che *shippa* software funzionante.
